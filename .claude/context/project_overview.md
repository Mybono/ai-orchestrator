# Project Overview

_Last updated: 2026-04-03 by planner after task: create project_overview.md via full codebase exploration_

## Language(s)

- Shell (Bash): `install.sh` — no standards file; script logic only
- Markdown: all agent, command, and skill files — no language standards apply
- Python (inline, inside agent heredocs): code generation snippets embedded in `.md` files — not a Python project per se, but Python 3 + `ollama` library is required at runtime

This is a **configuration/tooling repository** — it contains no application source code. All "code" is Markdown instructions for Claude agents and slash commands.

## Architecture & Conventions

- All agents are Markdown files in `agents/` with a YAML front-matter block (`name`, `description`, `model`, `tools`)
- All slash commands are Markdown files in `commands/` with no front-matter — they describe steps to orchestrate agents
- Language standards are in `skills/` and are named `<lang>-code-standarts.md` (note: "standarts" not "standards" — intentional spelling in filenames)
- Context files produced during a task go to `.claude/context/` — specifically `task_context.md`, `coder_output.md`, and `project_overview.md`
- The full pipeline is always: planner writes context → coder generates code → build/type check → reviewer(s) in parallel → fix loop (max 3 rounds)
- Code generation (coder, quick-coder, commit, doc-writer, test-agent) always calls Ollama via a Python heredoc, never inline — this avoids shell interpolation issues with file contents
- The `commit` agent is also available as a shell alias: `alias commit='claude -p "stage all changes and commit" --agent commit --model haiku --allowedTools "Bash"'`
- `install.sh` uses symlinks, not file copies — a `git pull` updates everything without reinstall
- `settings.json.template` uses `__HOME__` as a placeholder; `install.sh` substitutes it with `$HOME` via `sed`
- The planner agent checks for `project_overview.md` first (Phase 0 fast path) and skips full exploration if it exists

## Do Not Touch

- `README.md`: managed exclusively by the `doc-writer` agent — a `PreToolUse` hook in `settings.json` will block direct edits
- `docs/` (any path matching `/docs/`): same hook protection as README.md
- `.claude/settings.json`: generated by `install.sh` from the template — do not edit directly on each machine
- `skills/*-code-standarts.md` filenames: the "standarts" typo is load-bearing — all agents reference these exact filenames; renaming breaks detection

## Known Constraints

- Never mock Ollama in tests — tests that require Ollama must be skipped with `@pytest.mark.skipif(not ollama_available(), reason="Ollama not running")`
- The `debug.md` agent file does not exist in `agents/` — it exists only as `commands/debug.md` (the `/debug` command has no separate agent, it runs inline)
- `quick-coder` uses `qwen2.5-coder:1.5b` with a hard 4096-token context limit — never use it for multi-file changes or new classes
- The fix loop in `/implement` is capped at 3 rounds — after that, unresolved issues are reported to the user, not silently ignored
- `doc-writer` uses `think=False` when calling `qwen3:8b` — do not remove this flag (avoids unnecessary chain-of-thought overhead)
- All Ollama calls use Python 3 + the `ollama` PyPI package — the system must have Python 3 and `pip install ollama` available
- `commit` agent stages with `git add -A` but must never include `venv/`, `dist/`, `*.egg-info/`, `__pycache__/`, `.env`, or cache files
